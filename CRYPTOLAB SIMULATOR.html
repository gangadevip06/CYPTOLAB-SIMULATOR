Classical · Substitution
Caesar Cipher
Shift Substitution — 50 BC
Each letter in the plaintext is shifted forward by a fixed number of positions in the alphabet. Julius Caesar famously used a shift of 3. Simple but historically significant — the foundation of modern substitution ciphers.
Classical · Polyalphabetic
Vigenère Cipher
Keyword Substitution — 1553
Uses a repeating keyword to apply multiple different Caesar shifts. Each letter of the key determines the shift for the corresponding plaintext letter. Called "le chiffre indéchiffrable" — the unbreakable cipher — for 300 years.
Classical · One-Time Pad
Vernam Cipher
XOR One-Time Pad — 1917
The only cipher proven to be perfectly secure (Shannon, 1949). Each character is XOR'd with a corresponding key character. The key must be truly random, same length as the message, and never reused. Impractical but theoretically unbreakable.
Classical · Linear Algebra
Hill Cipher
Matrix Encryption — 1929
Uses a key matrix to encrypt blocks of letters via linear algebra. Each block of plaintext is multiplied by the key matrix modulo 26. Decryption requires computing the matrix inverse mod 26 using the Extended Euclidean Algorithm.

Classical · Digraph
Playfair Cipher
5×5 Grid Substitution — 1854
Uses a 5×5 key square to encrypt pairs of letters (digraphs). I and J share one cell. Three rules: same row → shift right, same column → shift down, rectangle → swap columns. First practical digraph cipher, used by the British in WWI.

Modern · Symmetric · Block Cipher
AES — Advanced Encryption Standard
Rijndael · NIST Standard 2001 · 128/192/256-bit
AES processes data in 4×4 byte blocks (128-bit). It applies 10/12/14 rounds for 128/192/256-bit keys, respectively. Each round: SubBytes → ShiftRows → MixColumns → AddRoundKey. Currently the world's most widely used symmetric encryption standard.
Modern · Symmetric · Feistel Network
DES — Data Encryption Standard
IBM / NIST · 1977 · 56-bit Effective Key
DES uses a 64-bit key (56-bit effective) and processes 64-bit blocks through 16 Feistel rounds. Each round: split L/R halves → expand R (32→48 bits) → XOR with subkey → S-box substitution → permute → XOR with L. Now considered insecure; replaced by AES.
Modern · Asymmetric · Public-Key
RSA — Rivest–Shamir–Adleman
MIT · 1977 · Integer Factorization Problem
RSA security relies on the difficulty of factoring large numbers. Key generation: pick primes p, q → n=p·q → φ(n)=(p-1)(q-1) → choose e coprime to φ(n) → find private key d via Extended Euclidean Algorithm. Encrypt: C=M^e mod n. Decrypt: M=C^d mod n.
Modern · Key Exchange · Discrete Log
Diffie–Hellman Key Exchange
Stanford · 1976 · Discrete Logarithm Problem
Alice and Bob agree on public values p (prime) and g (generator). Each picks a secret private key, computes a public key, exchanges it publicly, then independently computes the same shared secret — without ever transmitting the secret over the network.
Hash Function · 128-bit · Broken
MD5 — Message Digest Algorithm 5
Ron Rivest · MIT · 1992 · 32 Hex Characters
MD5 produces a 128-bit (32 hex character) hash. It processes data in 512-bit blocks through 4 rounds of 16 operations each (64 total). Uses bitwise operations, modular addition, and non-linear functions F, G, H, I. ⚠ MD5 is now cryptographically broken — collision attacks exist.
Hash Function · 256-bit · Secure
SHA-256 — Secure Hash Algorithm 256
NSA / NIST · 2001 · SHA-2 Family · 64 Hex Characters
SHA-256 produces a 256-bit (64 hex character) hash. Processes 512-bit blocks through 64 rounds using 8 working variables (a-h), message schedule expansion W[0..63], and bitwise operations (Ch, Maj, Σ0, Σ1, σ0, σ1) with 64 fixed round constants K[0..63].
